## Overview

a digital project monitoring application designed to streamline communication and progress reporting between housing developers and construction foremen. By leveraging GPS-enabled photo documentation and role-based access, the platform replaces inefficient manual reporting with a transparent, real-time tracking system to ensure projects stay on schedule.

## Key Features

- **Role-Based Access Control**: Designed specialized interfaces for Developers and Foremen (Mandor), ensuring strict data separation and workflows based on their specific on-site responsibilities.

-**Interactive Project Creation & Mapping**: Developed a comprehensive project creation tool for Developers, integrated with a mapping feature to precisely define construction locations and spatial scopes.

-**Real-Time Progress Reporting**: Empowered Foremen with a streamlined reporting module to submit construction progress updates directly from the site, keeping Developers consistently informed.

-**Centralized Workforce Management**: Provided Developers with an administrative dashboard to easily register new Foreman accounts and monitor the complete directory of active personnel.

-**Project Lifecycle Tracking**: Created a dedicated system for Foremen to access their currently assigned projects and review a complete history of successfully finished constructions.

-**Unified Profile Management**: Implemented customized profile settings for both Developers and Foremen to maintain accurate professional information and contact details.

## Technologies Used

- **Figma**: For designing intuitive user interfaces, mapping out user journeys, and creating prototypes before the development phase.
- **Flutter**: For building a responsive, cross-platform mobile application tailored for both on-site foremen and developers.
- **Dart**: For structuring the frontend logic and powering the Flutter framework to ensure smooth user interactions and fast rendering.
- **C#**: For developing robust and secure backend RESTful APIs to handle complex business logic and system communication.
- **PostgreSQL**: For serving as a reliable relational database to securely manage user roles, project records, spatial data, and progress reports.

## Challenges and Learnings

The challenges in developing DevTrack was establishing a seamless and secure integration between the C# RESTful API and the Flutter mobile application. Implementing a robust authentication system was particularly complex, as it required securely managing user sessions and role-based access (Developer vs. Foreman) using JWT (JSON Web Tokens). Furthermore, ensuring the API logic accurately mapped to the PostgreSQL database structure—while maintaining fast and reliable data synchronization—required meticulous architectural planning.

## Outcome
The implementation of DevTrack has transformed construction project monitoring by establishing a direct, real-time communication channel between developers and on-site foremen. With integrated spatial mapping and instant progress reporting, management now has complete, transparent visibility over site activities, drastically reducing miscommunication and reporting delays. This project has proven the value of digitalizing on-site workflows to ensure construction projects are tracked accurately and managed efficiently from start to finish.
